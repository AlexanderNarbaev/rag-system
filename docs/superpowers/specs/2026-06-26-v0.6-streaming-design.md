# v0.6 — Real-Time Indexing & Streaming Design Spec

> Date: 2026-06-26 | Status: Approved

## Scope
Eliminate ETL latency with streaming ingestion. Optimize proxy performance for lower latency and higher throughput.

## Features

### A. Streaming ETL (3 features)
1. **Webhook-driven ingestion** — Confluence/GitLab webhooks → Redis Streams → incremental indexing pipeline
2. **Streaming ETL pipeline** — Redis Streams consumer replacing batch scheduler
3. **Live Qdrant upserts** — atomic chunk-level updates with zero downtime

### B. Proxy Performance (3 features)
4. **Streaming LLM optimization** — SSE improvements for TTFT < 1s
5. **Model warm-up endpoint** — pre-load embedder/reranker/LLM on startup
6. **Response compression** — gzip/brotli middleware for 60%+ reduction

### C. Documentation
Update all docs to reflect v0.6 features.

## Principles
- Air-gapped first
- Graceful degradation (Redis unavailable → fall back to batch)
- TDD: test → code → commit → push
- All config via environment variables
- English code/comments
