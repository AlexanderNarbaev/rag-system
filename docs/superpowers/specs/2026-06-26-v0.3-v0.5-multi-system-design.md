# v0.3–v0.5 Multi-System Design Spec

> Date: 2026-06-26 | Approved: ✅

## Scope

Consolidated implementation of roadmap v0.3 (Token Optimization & Quality), v0.4 (Security & RBAC), v0.5 (Multi-Modal RAG), plus research-driven self-correction improvements.

## Subsystems

### A. Token Optimization & Quality (v0.3)
- Retrieval evaluation dataset + pipeline (MRR, Recall@k, nDCG)
- Context grounding score (cosine-similarity)
- Token optimizer integration into context builder
- Dynamic top-k via SLM query complexity scoring
- vLLM prefix caching config
- Custom exception hierarchy (RAGError, RetrievalError, LLMError)
- Readiness/liveness probes (/v1/health/live, /v1/health/ready)
- Dependency scanning (pip-audit in CI)

### B. Self-Correction Engine
- NLI-based grounding checker (entailment per claim)
- CRAG knowledge strip decomposition (decompose → filter → recompose)
- Self-critique node in LangGraph (SLM rates usefulness → retry if low)
- LLMLingua-style token compression (SLM perplexity-based pruning)
- LongContextReorder (best chunks at start/end of prompt)
- Confidence threshold calibration
- Query complexity scoring (1-10)

### C. Security & RBAC (v0.4)
- JWT authentication (Keycloak OIDC)
- RBAC (admin/expert/user/read-only)
- User data isolation (per-namespace Qdrant filtering)
- Input sanitization
- Audit logging with user identity
- HTTPS/TLS nginx reverse proxy config
- Log rotation (100MB, 7-day retention)
- A/B test harness

### D. Multi-Modal RAG (v0.5)
- Image embedding & retrieval (CLIP/BLIP)
- Code-aware chunking (AST-based)
- Table extraction from Confluence/Jira
- Multi-modal context assembly
- ColBERT late interaction (bge-m3 multi-vectors)
- Automated cold storage cleanup
- Reranker fine-tuning from HITL feedback
- Full documentation update

## Architecture Principles
- Air-gapped first
- Graceful degradation — every component optional
- TDD: test first, then implementation
- Every step: commit + push
- English for code/comments
