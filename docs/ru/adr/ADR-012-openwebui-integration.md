# ADR-012: Архитектура интеграции с OpenWebUI

**Статус:** Принято
**Дата:** 2026-07-10
**Автор:** Architecture Design
**Область:** Интеграция OpenWebUI с RAG-прокси, хранилищем файлов MinIO и выделенной векторной БД

---

## Контекст

RAG-системе нужен удобный веб-интерфейс для нетехнических пользователей. OpenWebUI (ранее Ollama WebUI) предоставляет:

- Чат-интерфейс с выбором модели
- Загрузку и управление файлами
- Поддержку вызова инструментов/функций
- Управление базой знаний
- Аутентификацию пользователей и администрирование

Система должна работать в изолированной корпоративной среде с:

- Выделенным экземпляром MinIO для хранения файлов
- Общим Qdrant для векторного поиска
- RAG-прокси в качестве основного LLM-бэкенда

## Решение

Интегрировать OpenWebUI как интерфейс первого класса с:

1. **Подключение через OpenAI-совместимый API** — OpenWebUI подключается к RAG-прокси через
   `OPENAI_API_BASE_URL=http://rag-proxy:8080/v1`
2. **Хранилище S3/MinIO** — Загруженные файлы хранятся в MinIO через `STORAGE_PROVIDER=s3`
3. **Общий Qdrant** — Встроенная RAG-система OpenWebUI использует тот же экземпляр Qdrant
4. **Интеграция сервера инструментов** — RAG-прокси зарегистрирован как OpenAPI-сервер инструментов
5. **Выделенный Docker Compose** — `docker-compose.openwebui.yml` со всеми сервисами

## Архитектура

```
User Browser → OpenWebUI (3000) → RAG Proxy (8080) → LLM Backend (8000)
                     ↓
              MinIO (9000) ← File uploads
                     ↓
              Qdrant (6333) ← Vector search (shared)
```

## Последствия

### Положительные

- Нетехнические пользователи получают удобный чат-интерфейс
- Загрузка файлов обрабатывается MinIO (S3-совместимый, масштабируемый)
- Вызов инструментов работает через OpenAPI-интеграцию
- Общий Qdrant обеспечивает единые результаты поиска

### Отрицательные

- Две RAG-системы (OpenWebUI + прокси) могут путать пользователей
- Дополнительная инфраструктура (MinIO, контейнер OpenWebUI)
- Поток загрузки файлов требует синхронизации с ETL-пайплайном

### Смягчения

- Чётко документировать, какую RAG-систему использовать для какой задачи
- Использовать отдельные коллекции Qdrant для OpenWebUI и прокси-RAG
- Создать скрипт синхронизации загрузок OpenWebUI → входные данные ETL
