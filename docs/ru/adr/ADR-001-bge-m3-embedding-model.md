# ADR-001: Выбор BAAI/bge-m3 в качестве модели эмбеддингов

**Статус:** Принято
**Дата:** 2026-06-22

## Контекст

RAG-системе требуется модель эмбеддингов, способная создавать как плотные, так и разреженные векторы для гибридного поиска, поддерживающая мультиязычные технические документы (русский и английский) из Confluence, Jira и GitLab. Среда является автономной (air-gapped) — модели должны быть загружены один раз и работать офлайн. Ключевые требования: контекст 8192 токенов (технические документы часто превышают 512 токенов), выдача dense + sparse за один проход и качество кросс-языкового поиска.

Рассмотренные альтернативы: `text-embedding-3-small` (API OpenAI — требует интернет, нет разреженных векторов), `e5-mistral-7b-instruct` (большая модель, больше VRAM, нет нативных sparse), `multilingual-e5-large` (только dense), и `sentence-transformers/all-MiniLM-L6-v2` (ограничение 512 токенов, только английский).

## Решение

**Использовать `BAAI/bge-m3` в качестве единственной модели эмбеддингов.** Она предоставляет плотные (1024-dim), разреженные (лексические BM25-стиль) и многомерные представления в стиле ColBERT за один вызов encode. Модель поддерживает более 100 языков и контекст ввода 8192 токенов.

Реализация в коде:
- `proxy/app/retrieval.py:64` загружает `SentenceTransformer(EMBEDDER_MODEL, device=EMBEDDER_DEVICE)` с `BAAI/bge-m3` по умолчанию.
- `etl/indexer/qdrant_hybrid.py:89` загружает ту же модель для индексации ETL с использованием `encode_sparse()` в `qdrant_hybrid.py:152` для извлечения разреженных векторов.
- Плотные векторы используются для семантического косинусного поиска; разреженные векторы — для лексического BM25 по той же коллекции с RRF-фьюжном в `retrieval.py:113`.

## Последствия

**Положительные:** Единая модель для индексации и поиска во время запроса; не требуется синхронизация между отдельными dense/sparse моделями. Разреженный векторный индекс хранится на диске (`qdrant_hybrid.py:118-121`), что снижает использование памяти. Совместимость с air-gapped — веса модели загружаются один раз в общий том.

**Отрицательные:** Модель (~2.2 GB) больше чисто плотных альтернатив, таких как `all-MiniLM-L6-v2` (~90 MB). Разреженные векторы увеличивают объём коллекции на ~30%. Многовекторные представления ColBERT в настоящее время не используются в пайплайне (отложенная оптимизация).

**Смягчения:** Эмбеддинги кэшируются через Redis/in-memory (`retrieval.py:88-95`), снижая повторные вычисления. Инференс на CPU приемлем для пакетного пути ETL; ускорение на GPU опционально через `EMBEDDER_DEVICE=cuda` (`config.py:22`).
