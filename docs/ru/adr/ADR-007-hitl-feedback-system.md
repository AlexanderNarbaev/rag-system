# ADR-007: Система обратной связи Human-in-the-Loop

**Статус:** Принято
**Дата:** 2026-06-22

## Контекст

Качество ответов RAG-системы зависит от релевантности поиска, сборки контекста и точности генерации LLM. Без сбора
обратной связи проблемы качества остаются незамеченными, и отсутствует механизм непрерывного улучшения. Экспертам (
специалистам предметной области) нужен способ исправлять неточные ответы и отмечать хорошие. Кроме того, исправленные
ответы формируют ценный обучающий датасет для будущего файн-тюнинга языковой модели.

Рассмотренные альтернативы: **без обратной связи** (качество со временем ухудшается), **автоматизированная оценка** (
метрики типа ROUGE/BERTScore не отражают фактическую корректность для технического контента), **внешняя платформа
аннотаций** (избыточно, добавляет сложность интеграции).

## Решение

**Реализовать лёгкую систему обратной связи Human-in-the-Loop (HITL) с журналированием JSONL, экспертной панелью
Streamlit и экспортом обучающих датасетов.**

`InteractionLogger` (`proxy/app/hitl.py:28-109`) записывает каждую пару запрос-ответ в формате JSON Lines (
`interactions.jsonl`) с ID запроса, временной меткой, запросом пользователя (до 5000 символов контекста), ответом и
метаданными (модель, версия, IP клиента, статус кэша). Обратная связь хранится отдельно в `feedback.jsonl` (
`hitl.py:72-96`) с тремя типами: `positive`, `negative` и `correction` (экспертно исправленный ответ).

Логирование асинхронное и неблокирующее (`hitl.py:122-144`) — выполняется через `asyncio.to_thread()`, поэтому никогда
не задерживает ответ API. Может быть отключено через `LOG_REQUESTS=false` (`config.py:63`).

Экспортёр обучающих датасетов (`hitl.py:166-192`) фильтрует взаимодействия для:

1. Экспертно исправленных ответов (пара: исходный вопрос → исправленный ответ)
2. Положительно оценённых ответов (пара: вопрос → ответ системы)

Вывод в формате JSONL для пайплайнов файн-тюнинга (`{"prompt": "...", "completion": "..."}`).

Экспертная панель проверки (в `hitl_dashboard/`, на базе Streamlit) предоставляет:

- Просмотр недавних взаимодействий с поиском/фильтрацией
- Отправку исправлений и обратной связи
- Экспорт обучающих датасетов по требованию

## Последствия

**Положительные:** Нулевое влияние на задержку ответов API (асинхронное, неблокирующее логирование). Полный аудиторский
след всех взаимодействий позволяет отлаживать сбои поиска. Исправления со временем накапливаются в доменно-специфичный
датасет для файн-тюнинга. Формат JSONL прост, только для добавления и легко парсится стандартными инструментами.

**Отрицательные:** Файлы JSONL растут неограниченно — нет автоматической ротации или политики хранения. Обратная связь
зависит от доступности экспертов; без экспертов собираются только сырые логи (без исправлений, без положительных
сигналов). Нет аутентификации/авторизации на пути обратной связи — любой клиент может отправить отзыв.

**Смягчения:** Контекст обрезается до 5000 символов (`hitl.py:56`) для ограничения роста файлов логов. Экспорт обучающих
датасетов поддерживает `min_length=50` для фильтрации шума. Будущие улучшения: ротация логов через logrotate или
внутреннюю политику хранения, аутентификация эндпоинта обратной связи.
