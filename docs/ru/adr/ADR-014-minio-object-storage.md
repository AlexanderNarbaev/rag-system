# ADR-014: Объектное хранилище MinIO для управления файлами

**Статус:** Принято
**Дата:** 2026-07-10
**Автор:** Architecture Design
**Область:** S3-совместимое объектное хранилище для загрузки файлов, управления документами и артефактами моделей

---

## Контекст

RAG-системе нужно постоянное хранилище файлов для:

- Документов, загруженных пользователями (через OpenWebUI или API)
- Артефактов ETL-пайплайна (чанки, эмбеддинги)
- Артефактов моделей (LoRA-адаптеры, файн-тюнингованные модели)
- Файлов резервных копий

Требования:

- S3-совместимый API (для совместимости с boto3)
- Развёртывание в изолированной среде (без внешнего облачного хранилища)
- Масштабируемость и надёжность
- Интеграция с загрузкой файлов OpenWebUI

## Решение

Развернуть MinIO в качестве бэкенда объектного хранилища:

1. **S3-совместимый API** — Работает с boto3, AWS SDK и OpenWebUI
2. **Три бакета**:
    - `rag-documents` — Документы, загруженные пользователями
    - `rag-artifacts` — Артефакты моделей и данные для обучения
    - `open-webui` — Загрузки файлов OpenWebUI
3. **Развёртывание в Docker** — Одиночный контейнер с постоянным томом
4. **Предподписанные URL** — Безопасный временный доступ к файлам

## Архитектура

```
OpenWebUI → MinIO (9000) → S3 API
ETL Pipeline → MinIO → rag-documents bucket
Model Evolution → MinIO → rag-artifacts bucket
RAG Proxy → MinIO → File metadata + presigned URLs
```

## Эндпоинты API

| Эндпоинт                       | Метод               | Описание                               |
|--------------------------------|---------------------|----------------------------------------|
| `POST /v1/files`               | Загрузка файла      | Мultipart-загрузка в MinIO             |
| `GET /v1/files`                | Список файлов       | Перечисление объектов в бакете         |
| `GET /v1/files/{id}`           | Метаданные          | Метаданные объекта                     |
| `GET /v1/files/{id}/download`  | Скачивание          | Потоковая передача содержимого объекта |
| `GET /v1/files/{id}/presigned` | Предподписанный URL | Временная ссылка для скачивания        |
| `DELETE /v1/files/{id}`        | Удаление            | Удаление объекта                       |

## Конфигурация

```env
MINIO_ENDPOINT=localhost:9000
MINIO_ACCESS_KEY=minioadmin
MINIO_SECRET_KEY=minioadmin
MINIO_BUCKET=rag-documents
MINIO_SECURE=false
```

## Последствия

### Положительные

- S3-совместимый API работает со всеми AWS SDK
- Изолированность (нет внешних зависимостей)
- Масштабируемость (можно добавить узлы в будущем)
- Нативная поддержка OpenWebUI

### Отрицательные

- Дополнительный компонент инфраструктуры
- Накладные расходы на управление хранилищем
- Сложность резервного копирования

### Смягчения

- Одиночный Docker-контейнер для небольших развёртываний
- Автоматизированные скрипты резервного копирования в `scripts/ops/`
- Мониторинг через проверку состояния
