## FR-32. HyDE — Hypothetical Document Embeddings

**Описание:**
Перед поиском система генерирует «гипотетический документ» — текст, который мог бы
ответить на вопрос. Эмбеддинг этого документа используется для поиска вместо (или
в дополнение к) эмбеддингу самого вопроса. Это улучшает sparse-поиск, потому что
гипотетический документ содержит больше релевантных терминов.

**Критерий приёмки:**
1. HyDE-поиск возвращает результаты, отличные от обычного поиска
2. Объединение результатов (HyDE + обычный) даёт более полный контекст
3. Лог содержит "HyDE expansion: N additional results"

**Статус:** ⚠️ Код есть (`proxy/app/core/hyde.py`), нужен интеграционный тест
**Приоритет:** HIGH
**Связь:** rag-maturity-assessment L5.3

## FR-33. CRAG — Corrective Retrieval-Augmented Generation

**Описание:**
После поиска система оценивает качество результатов по 4 факторам:
- Score distribution (0.4) — распределение скоров релевантности
- Coverage ratio (0.3) — покрытие аспектов запроса
- Result count (0.2) — количество найденных результатов
- Recency decay (0.1) — свежесть документов

На основе оценки система решает: USE (использовать), REWRITE (переписать запрос),
EXPAND (расширить поиск), FALLBACK (не отвечать).

**Критерий приёмки:**
1. Высокий confidence → USE → ответ генерируется из найденного контекста
2. Низкий confidence → REWRITE → запрос переписывается и поиск повторяется
3. Очень низкий confidence → FALLBACK → система сообщает о недостаточности
4. Лог содержит "Retrieval quality: confidence=X.XXX, action=USE/REWRITE/EXPAND/FALLBACK"

**Статус:** ⚠️ Код есть (`proxy/app/core/retrieval_evaluator.py`), нужен интеграционный тест
**Приоритет:** HIGH
**Связь:** rag-maturity-assessment L5.1

## FR-34. Self-reflection (post-generation critique)

**Описание:**
После генерации ответа система отправляет ответ обратно в LLM с промптом:
«Оцени, подтверждается ли этот ответ контекстом. Оценка 0-10.» Если оценка < 5 —
ответ помечается как ненадёжный.

**Критерий приёмки:**
1. Ответ, подтверждённый контекстом — self-reflection score ≥ 5
2. Ответ, не подтверждённый контекстом — self-reflection score < 5, лог: "Self-critique failed"
3. Self-reflection score записывается в метрики

**Статус:** ⚠️ Код есть (`proxy/app/core/confidence.py`), нужен интеграционный тест
**Приоритет:** HIGH
**Связь:** rag-maturity-assessment L5.4

## FR-35. NLI-based answer grounding

**Описание:**
Система проверяет «обоснованность» ответа с помощью:
1. Косинусного сходства между эмбеддингом ответа и эмбеддингом контекста
2. NLI-классификации (entailment/contradiction/neutral)

Ответы с grounding score < 0.70 помечаются для review.

**Критерий приёмки:**
1. Ответ, следующий из контекста — grounding score ≥ 0.70
2. Ответ, противоречащий контексту — grounding score < 0.70, помечен для review
3. Лог содержит "Grounding score: X.XXX"

**Статус:** ⚠️ Код есть (`proxy/app/core/grounding.py`), нужен интеграционный тест
**Приоритет:** HIGH
**Связь:** rag-maturity-assessment L5.5

## FR-36. Детекция галлюцинаций

**Описание:**
Система анализирует ответ на наличие утверждений, не поддерживаемых контекстом.
Каждое утверждение проверяется отдельно. Неподдерживаемые утверждения флагируются.

**Критерий приёмки:**
1. Ответ с галлюцинациями — hallucination_score > 0 (есть unsupported claims)
2. Ответ без галлюцинаций — hallucination_score = 0
3. Флагированные утверждения записываются в лог

**Статус:** ⚠️ Код есть (`proxy/app/core/hallucination.py`), нужен интеграционный тест
**Приоритет:** HIGH
**Связь:** rag-maturity-assessment L5.5

## FR-37. Corrective re-generation

**Описание:**
Если ответ не прошёл проверку (низкий confidence, grounding score, или обнаружены
галлюцинации), система:
1. Расширяет контекст (ищет больше чанков)
2. Изменяет промпт (добавляет «отвечай только на основе контекста»)
3. Снижает temperature (для более детерминированного ответа)
4. Генерирует ответ заново

**Критерий приёмки:**
1. Непрошедший проверку ответ — trigger re-generation
2. Re-generated ответ — проходит проверку (или система сообщает о неудаче)
3. Максимум 2 попытки re-generation

**Статус:** ⚠️ Код есть (`proxy/app/core/confidence.py`), нужен интеграционный тест
**Приоритет:** HIGH
**Связь:** rag-maturity-assessment L5.6

## FR-38. LLMLingua token-level compression ❌

**Описание:**
Система сжимает контекст на уровне токенов, удаляя «малозначимые» токены с
сохранением ключевой информации. Целевое сжатие: 2-5×, потеря информации < 5%.

**Критерий приёмки:**
1. Сжатый контекст в 2-5 раз короче оригинала
2. Ключевые факты сохранены (проверка: LLM отвечает корректно на сжатом контексте)
3. Latency сжатия < 100ms для контекста 10K токенов

**Статус:** ❌ Нужна реализация
**Приоритет:** HIGH
**Связь:** rag-maturity-assessment L5

## FR-39. LongContextReorder ❌

**Описание:**
Для борьбы с эффектом «lost in the middle» (LLM хуже запоминает информацию в середине
контекста) система переупорядочивает чанки: самые релевантные — в начало и конец,
менее релевантные — в середину.

**Критерий приёмки:**
1. Самый релевантный чанк — первый в контексте
2. Второй по релевантности — последний
3. Остальные — в середине, отсортированные по релевантности

**Статус:** ❌ Нужна реализация
**Приоритет:** HIGH
**Связь:** rag-maturity-assessment L5
