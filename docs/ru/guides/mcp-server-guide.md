# Руководство по MCP-серверу

**Статус реализации:** Реализовано. MCP-сервер обеспечивает интеграцию с IDE через Model Context Protocol, предоставляя
RAG-инструменты для OpenCode, Claude Desktop и других MCP-совместимых клиентов.

---

## 1. Что такое MCP (Model Context Protocol)

Model Context Protocol (MCP) — это открытый протокол, позволяющий AI-ассистентам и инструментам IDE взаимодействовать с
внешними источниками данных и инструментами через стандартизированный интерфейс. Вместо создания индивидуальных
интеграций для каждого AI-инструмента MCP предоставляет единый сервер, к которому может подключиться любой совместимый
клиент.

**Ключевые концепции:**

- **Инструменты (Tools)** — вызываемые функции, которые AI может использовать (например, поиск по базе знаний)
- **Ресурсы (Resources)** — данные, которые AI может читать (например, содержимое документов, конфигурация)
- **Подсказки (Prompts)** — предопределённые шаблоны промптов для типовых задач
- **Транспорты (Transports)** — методы коммуникации (STDIO для локальных, HTTP для удалённых)

MCP-сервер RAG-системы предоставляет возможности поиска и извлечения из базы знаний в виде MCP-инструментов, позволяя
AI-ассистентам, интегрированным в IDE, искать корпоративную документацию, получать конкретные документы и提供
контекстно-зависимые ответы.

## 2. Настройка MCP-сервера

### 2.1 Предварительные требования

```bash
# Установка зависимостей (включены в требования прокси)
pip install fastmcp

# Проверка наличия модуля сервера
python -c "import mcp_server.server; print('MCP server module OK')"
```

### 2.2 Запуск MCP-сервера

**STDIO транспорт** (для локальной интеграции с IDE):

```bash
# MCP-клиент запускает сервер автоматически
python -m mcp_server.server
```

**Streamable HTTP транспорт** (для удалённого/общего доступа):

```bash
# Запуск как HTTP-сервер
python -m mcp_server.server --transport http --port 8081
```

### 2.3 Конфигурация

MCP-сервер читает конфигурацию из переменных окружения:

| Переменная      | По умолчанию               | Описание                                      |
|-----------------|----------------------------|-----------------------------------------------|
| `RAG_PROXY_URL` | `http://localhost:8080/v1` | Эндпоинт API RAG-прокси                       |
| `MCP_TRANSPORT` | `stdio`                    | Тип транспорта: `stdio` или `http`            |
| `MCP_PORT`      | `8081`                     | HTTP-порт (при использовании HTTP транспорта) |
| `MCP_HOST`      | `0.0.0.0`                  | HTTP адрес привязки                           |

## 3. Доступные инструменты

MCP-сервер предоставляет следующие инструменты подключённым клиентам:

### 3.1 `search_knowledge_base`

Гибридный поиск (dense + sparse с RRF-слиянием) по всем индексированным документам.

| Параметр    | Тип       | Обязательный | Описание                                 |
|-------------|-----------|--------------|------------------------------------------|
| `query`     | `string`  | да           | Текст поискового запроса                 |
| `top_k`     | `integer` | нет          | Количество результатов (по умолчанию: 5) |
| `namespace` | `string`  | нет          | Фильтр по пространству имён              |
| `version`   | `string`  | нет          | Фильтр по версии документа               |

**Пример использования в IDE:**
> "Найди в базе знаний решения по архитектуре аутентификации"

### 3.2 `get_document`

Получить конкретный документ по его ID источника.

| Параметр | Тип      | Обязательный | Описание                                  |
|----------|----------|--------------|-------------------------------------------|
| `doc_id` | `string` | да           | ID документа (хеш чанка или ID источника) |

**Пример использования в IDE:**
> "Получи документ с ID CONFL-12345"

### 3.3 `list_sources`

Список доступных источников данных с количеством документов.

Параметры не требуются.

**Пример использования в IDE:**
> "Какие источники данных доступны в базе знаний?"

### 3.4 `get_graph_context`

Получить контекст графа Neo4j для сущности (требуется `GRAPH_ENABLED=true`).

| Параметр | Тип       | Обязательный | Описание                               |
|----------|-----------|--------------|----------------------------------------|
| `entity` | `string`  | да           | Имя сущности для поиска                |
| `depth`  | `integer` | нет          | Глубина обхода графа (по умолчанию: 2) |

**Пример использования в IDE:**
> "Какой контекст графа вокруг сущности 'AuthService'?"

### 3.5 `submit_feedback`

Отправить экспертную обратную связь для улучшения качества HITL.

| Параметр      | Тип      | Обязательный | Описание                                |
|---------------|----------|--------------|-----------------------------------------|
| `feedback_id` | `string` | да           | ID обратной связи из предыдущего ответа |
| `rating`      | `string` | да           | `positive` или `negative`               |
| `correction`  | `string` | нет          | Исправленный текст ответа               |

### 3.6 `get_confidence`

Получить оценку достоверности для пары запрос-ответ.

| Параметр | Тип      | Обязательный | Описание              |
|----------|----------|--------------|-----------------------|
| `query`  | `string` | да           | Исходный запрос       |
| `answer` | `string` | да           | Сгенерированный ответ |

## 4. Доступные ресурсы

MCP-сервер также предоставляет ресурсы только для чтения:

| URI ресурса    | Описание                                                               |
|----------------|------------------------------------------------------------------------|
| `rag://config` | Текущая конфигурация прокси (несекретная)                              |
| `rag://health` | Статус здоровья системы                                                |
| `rag://stats`  | Статистика базы знаний (количество документов, разбивка по источникам) |

## 5. Использование с OpenCode

### 5.1 Конфигурация

Добавьте MCP-сервер в ваш `opencode.json`:

```json
{
  "providers": {
    "rag-system": {
      "name": "RAG System",
      "base_url": "http://localhost:8080/v1",
      "api_key": "${RAG_API_KEY}",
      "models": ["your-model-name"]
    }
  },
  "mcp_servers": {
    "rag-system": {
      "type": "streamableHttp",
      "url": "http://localhost:8081/mcp",
      "description": "RAG System — поиск по корпоративной базе знаний"
    }
  },
  "model": "rag-system/your-model-name"
}
```

### 5.2 STDIO транспорт (локальный)

Для локальной разработки, где MCP-сервер запускается на той же машине:

```json
{
  "mcp_servers": {
    "rag-system": {
      "type": "stdio",
      "command": "python",
      "args": ["-m", "mcp_server.server"],
      "env": {
        "RAG_PROXY_URL": "http://localhost:8080/v1"
      }
    }
  }
}
```

### 5.3 Примеры использования

После настройки OpenCode может напрямую использовать RAG-инструменты:

- **Поиск:** "Найди в базе знаний архитектуру аутентификации"
- **Получение:** "Получи документ с ID источника CONFL-12345"
- **Список:** "Перечисли все доступные источники данных"
- **Граф:** "Какой контекст графа вокруг сущности 'AuthService'?"
- **Обратная связь:** "Отправь положительную обратную связь для этого ответа"

## 6. Использование с Claude Desktop

### 6.1 Конфигурация

**macOS:** Отредактируйте `~/Library/Application Support/Claude/claude_desktop_config.json`

**Windows:** Отредактируйте `%APPDATA%\Claude\claude_desktop_config.json`

**STDIO транспорт:**

```json
{
  "mcpServers": {
    "rag-system": {
      "command": "python",
      "args": ["-m", "mcp_server.server"],
      "env": {
        "RAG_PROXY_URL": "http://localhost:8080/v1"
      }
    }
  }
}
```

**HTTP транспорт (удалённый сервер):**

```json
{
  "mcpServers": {
    "rag-system": {
      "url": "http://rag-proxy.internal.company.com:8081/mcp"
    }
  }
}
```

### 6.2 Использование

После перезапуска Claude Desktop RAG-инструменты появятся в меню инструментов. Claude может:

1. Искать в корпоративной базе знаний при ответах на вопросы
2. Получать конкретные документы для подробной информации
3. Исследовать связи сущностей через граф знаний
4. Отправлять обратную связь для улучшения качества ответов

## 7. Устранение неполадок

### 7.1 Сервер не запускается

```bash
# Проверка импорта модуля
python -c "import mcp_server.server; print('OK')"

# Проверка установки fastmcp
pip show fastmcp

# Запуск с подробным логированием
MCP_LOG_LEVEL=DEBUG python -m mcp_server.server
```

### 7.2 Клиент не может подключиться

**STDIO транспорт:**

- Проверьте, что `command` и `args` указаны правильно в конфигурации клиента
- Убедитесь, что путь к Python указан верно (используйте полный путь при необходимости)
- Убедитесь, что `RAG_PROXY_URL` доступен из процесса MCP-сервера

**HTTP транспорт:**

```bash
# Тест HTTP-эндпоинта напрямую
curl http://localhost:8081/mcp

# Проверка, что порт открыт
netstat -tlnp | grep 8081
```

### 7.3 Инструменты возвращают ошибки

```bash
# Проверка здоровья RAG-прокси
curl http://localhost:8080/v1/health

# Проверка, что инструменты включены
curl http://localhost:8080/v1/tools

# Просмотр логов прокси
docker logs rag-proxy --tail 50
```

### 7.4 Контекст графа недоступен

Инструмент `get_graph_context` требует включённого Neo4j:

```bash
# Проверка, что Neo4j запущен
docker ps | grep neo4j

# Проверка конфигурации графа
grep GRAPH_ENABLED proxy/.env
# Должно быть: GRAPH_ENABLED=true
```

### 7.5 Частые сообщения об ошибках

| Ошибка                | Причина                 | Решение                                                       |
|-----------------------|-------------------------|---------------------------------------------------------------|
| `Connection refused`  | Прокси не запущен       | Запустите прокси: `docker-compose up -d`                      |
| `Tool not found`      | Инструменты не включены | Установите `TOOLS_ENABLED=true` в `.env`                      |
| `401 Unauthorized`    | Несовпадение API ключа  | Проверьте, что `RAG_API_KEY` совпадает с конфигурацией прокси |
| `504 Gateway Timeout` | LLM-бэкенд медленный    | Увеличьте `REQUEST_TIMEOUT` в `.env`                          |
| `Neo4j not available` | Граф отключён           | Установите `GRAPH_ENABLED=true` и настройте Neo4j             |
