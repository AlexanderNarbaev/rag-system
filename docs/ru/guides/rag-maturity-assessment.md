## 1. Модель зрелости RAG

Модель зрелости RAG определяет пять прогрессивных уровней возможностей Retrieval-Augmented Generation. Каждый уровень
строится на предыдущем, добавляя новые измерения качества, интеллектуальности и самосознания.

### Обзор

| Уровень | Название            | Ключевые возможности                                                                           | Наш статус   |
|---------|---------------------|------------------------------------------------------------------------------------------------|--------------|
| 1       | **Naive RAG**       | Одиночный плотный поиск, без реранка, без дедупликации                                         | ✅ Превзойдён |
| 2       | **Advanced RAG**    | Гибридный (dense+sparse), cross-encoder реранк, дедупликация, фильтрация версий                | ✅ Реализован |
| 3       | **GraphRAG**        | Извлечение сущностей, граф знаний Neo4j, многошаговый обход, сущностно-ориентированный поиск   | ✅ Реализован |
| 4       | **Agentic**         | LangGraph оркестратор, многошаговые циклы поиска, оценка достаточности, переписывание запросов | ✅ Реализован |
| 5       | **Self-Correcting** | CRAG-оценщик, HyDE-генерация, саморефлексия, корректирующие циклы, заземление галлюцинаций     | ✅ Реализован |

**Новое в v2.0 — Self-Correcting RAG (Уровень 5):**

- **HyDE-расширение запросов** — генерация гипотетических документов из запросов для улучшения разреженного поиска
- **CRAG-оценщик** — многофакторная оценка качества с маппингом уверенности на действия
- **Циклы саморефлексии** — пост-генерационная критика: LLM перечитывает ответ с учётом контекста, выявляет
  несоответствия
- **Обнаружение и заземление галлюцинаций** — NLI-верификация: косинусное сходство + классификация entailment
- **Корректирующая ре-генерация** — ответы с низкой уверенностью регенерируются с расширенным контекстом
- **Агентный вызов инструментов** — живые запросы к Confluence/Jira/GitLab API через function calling
- **Мультиязычная поддержка** — полная интернационализация RU/EN/DE/FR/ZH с параметром `lang`
- **Коннекторы к живым источникам** — прямая интеграция с API систем-источников
- **Графовые паттерны саморефлексии** — обогащение Neo4j рёбрами валидации и оценками уверенности связывания сущностей
- **LLMLingua сжатие** — сжатие промптов на уровне токенов (2-5x коэффициент)
- **LongContextReorder** — переранжирование для борьбы с эффектом "lost in the middle"
- **E2E-тесты** — полностековые тесты с реальными Qdrant/Neo4j/LLM
- **Бенчмарки производительности** — нагрузочное тестирование на 10/50/100 пользователях
- **Chaos/отказоустойчивое тестирование** — инжекция отказов, проверка graceful degradation
- **Kubernetes Helm-чарт** — HPA, пробы, config maps, секреты
- **Grafana-дашборды** — обзор, качество поиска, инфраструктура
- **Правила алертов Prometheus** — уровни critical/warning/info
- **Определения SLI/SLO** — доступность 99.5%, p95 < 5s, ошибки < 1%
- **HA-развёртывание** — репликация Qdrant, кластер Neo4j, Redis Sentinel
- **Автоматизация бэкапов** — Qdrant/Neo4j/Redis → S3/MinIO
- **DR runbook** — 8 сценариев восстановления с пошаговыми процедурами

**Новое в v0.6 — возможности Agentic+:**

- **Индексация в реальном времени** — потоковый ETL на Redis Streams с приёмом через вебхуки (задержка <5s)
- **Прогрев моделей** — эндпоинт `/v1/admin/warmup` устраняет задержку холодного старта
- **Оптимизация SSE TTFT** — пул соединений, chunked transfer, уменьшенная буферизация (TTFT <1s кэш.)
- **Сжатие ответов** — middleware gzip/brotli (снижение 60%+, <5ms CPU)

### Фундаментальный вывод

Система успешно реализует все пять уровней зрелости RAG. Ключевое достижение v2.0 — замыкание цикла самокоррекции:
система теперь оценивает свои ответы, обнаруживает проблемы (галлюцинации, низкую уверенность, незаземлённые
утверждения) и итеративно улучшает их через ре-генерацию с расширенным контекстом, скорректированными параметрами или
промптами на фактическую точность. Критический самокорректирующий цикл:

```
Запрос → HyDE-расширение → CRAG-поиск → Генерация → Саморефлексия → NLI-заземление → Корректирующая ре-генерация (при необходимости) → Ответ
```

Этот цикл обеспечивает непрерывное улучшение качества с каждым запросом, переходя от статического поиска к
динамическому, самоосознанному синтезу знаний.
