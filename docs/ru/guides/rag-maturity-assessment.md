## 1. Модель зрелости RAG

| Уровень | Название | Ключевые возможности | Наш статус |
|---------|----------|---------------------|------------|
| 1 | **Naive** | Одиночный плотный поиск, без реранка, без дедупликации | ✅ Превзойдён |
| 2 | **Advanced** | Гибридный (dense+sparse), cross-encoder реранк, дедупликация, фильтрация версий | ✅ Реализован |
| 3 | **GraphRAG** | Извлечение сущностей, граф знаний Neo4j, многошаговый обход, сущностно-ориентированный поиск | ✅ Реализован |
| 4 | **Agentic** | Оркестратор LangGraph, многошаговые циклы поиска, оценка достаточности, переписывание запросов, расширение графа | ✅ Реализован |
| 5 | **Self-Correcting** | Оценщик поиска (CRAG-стиль), HyDE-генерация документов, саморефлексия, корректирующие циклы обратной связи, заземление галлюцинаций | 🟡 Частично |

### Детали уровней

#### Уровень 1 — Naive RAG
- Только плотный векторный поиск с BAAI/bge-m3
- Без реранкинга, без обнаружения дубликатов, без осведомлённости о версиях
- **Статус:** Превзойдён. Это был наш базовый уровень при создании, но все компоненты вышли за его пределы с первого дня.

#### Уровень 2 — Advanced RAG
- **Гибридный поиск:** плотные (1024-dim) + разреженные (lexical) векторы с RRF-фьюжном (`retrieval.py:113-128`)
- **Cross-encoder реранк:** `ms-marco-MiniLM-L-6-v2` оценивает top-N чанков (`rerank.py`)
- **Дедупликация:** SHA-256 контентно-адресуемое версионирование чанков (`hash_versioning.py:52-275`)
- **Фильтрация версий:** `FieldCondition` Qdrant по полю `version`, доступный как параметр API `rag_version`
- **Статус:** Полностью реализован и работает.

#### Уровень 3 — GraphRAG
- **Извлечение сущностей:** извлечение сущностей из технических документов на основе spaCy (`entity_extractor.py`)
- **Интеграция Neo4j:** граф связей сущностей с Cypher-запросами (`neo4j_loader.py`)
- **Многошаговый обход:** `graph_expand_query()` в `retrieval.py` обогащает контекст связанными сущностями
- **Схема сущностей:** `schema.yaml` определяет типы узлов и связей
- **Статус:** Реализован. Расширение графа опционально (`USE_GRAPH_EXPANSION=false` по умолчанию). Требует работающего Neo4j вместе с Qdrant.

#### Уровень 4 — Agentic RAG
- **Оркестратор LangGraph:** 7-узловой граф состояний: rewrite → retrieve → check_sufficiency → rerank → graph_expand → build_context → generate (`orchestrator.py`)
- **Оценка достаточности:** `check_sufficiency()` использует порог score (0.6) для запуска циклов переписывания (`orchestrator.py:127-146`)
- **Переписывание запросов:** SLM или LLM переписывает неоднозначные запросы для лучшего поиска (`orchestrator.py:53-80`)
- **Условные циклы:** До `MAX_RETRIEVAL_LOOPS=3` итераций, пока не будет найден достаточный контекст
- **Статус:** Реализован, но опционален (`USE_LANGGRAPH=false` по умолчанию). Добавляет 200-500ms накладных расходов на запрос. Fallback к линейному пайплайну при отключении.

#### Уровень 5 — Self-Correcting RAG (Частично)
- **CRAG-оценщик:** Базовая проверка достаточности по порогу score существует; оценка уверенности на основе LLM ещё не реализована
- **HyDE (Hypothetical Document Embeddings):** Не реализовано. Преобразование запроса в документ могло бы улучшить разреженный поиск для технических запросов.
- **Саморефлексия:** Не реализована. Нет оценки качества ответа или шага самокритики после генерации.
- **Заземление галлюцинаций:** Формула оценки заземления задокументирована (`performance-quality.md:182`), но ещё не подключена к конвейеру генерации.
- **Корректирующие циклы:** `MAX_RETRIEVAL_LOOPS` обеспечивает базовую коррекцию поиска; полная коррекция на уровне ответа с повторной генерацией отсутствует.
- **Пробел:** 4 из 5 компонентов Уровня 5 частично специфицированы, но не реализованы в коде.

## 4. Бенчмарки качества поиска

### Целевые метрики

| Метрика | Определение | Цель | Критичность |
|--------|-----------|------|-------------|
| **MRR** | Mean Reciprocal Rank первого релевантного чанка | > 0.80 | Обязательно |
| **Recall@20** | Доля запросов с минимум 1 релевантным чанком в top-20 | > 0.90 | Обязательно |
| **nDCG@10** | Normalized Discounted Cumulative Gain для top-10 | > 0.85 | Желательно |
| **Precision@5** | Доля релевантных чанков в top-5 | > 0.70 | Желательно |
| **Hit Rate** | % запросов, возвращающих минимум 1 релевантный документ | > 0.95 | Обязательно |
| **Context Grounding** | Косинусное сходство между ответом и найденным контекстом | > 0.70 | Желательно |

### Оценка текущего состояния

| Метрика | Базовый (только dense) | Текущий (hybrid+rerank) | Цель | Пробел |
|--------|----------------------|------------------------|------|-------|
| MRR | ~0.62 (оценка) | **Не измерено** — нет оценочного датасета | > 0.80 | **Критический пробел** |
| Recall@20 | ~0.78 (оценка) | **Не измерено** | > 0.90 | **Критический пробел** |
| nDCG@10 | ~0.55 (оценка) | **Не измерено** | > 0.85 | **Критический пробел** |
| Precision@5 | Неизвестно | Неизвестно | > 0.70 | Неизвестно |
| Hit Rate | Неизвестно | Неизвестно | > 0.95 | Неизвестно |
| Context Grounding | Не реализовано | Не реализовано | > 0.70 | Не начато |

### Анализ пробелов

**Критические (блокируют production-развёртывание):**

1. **Нет размеченного оценочного датасета.** Документ `performance-quality.md` указывает "200+ пар запрос-документ" как цель, но этот датасет не существует. Без него качество поиска полностью анекдотично.
2. **Нет автоматизированного конвейера оценки.** Даже если бы датасет существовал, нет скрипта для вычисления метрик MRR, nDCG, Recall.
3. **Заземление контекста не реализовано.** Формула оценки заземления задокументирована (`performance-quality.md:182-189`), но код её не выполняет. Обнаружение галлюцинаций полностью отсутствует.

**Высокий приоритет:**

4. **Влияние реранкера не измерено.** Дельта `MRR_rerank − MRR_dense` никогда не вычислялась. Реранкер MiniLM-L-6-v2 может недостаточно помогать или перекомпенсировать — нет данных ни в ту, ни в другую сторону.
5. **Качество расширения графа неизвестно.** Нет измерения, улучшает или ухудшает ли связывание сущностей Neo4j качество ответов.
6. **Качество переписывания SLM не оценено.** Путь эвристического fallback имеет нулевую оценку качества.

**Средний приоритет:**

7. **Цикл обратной связи HITL не замкнут.** Исправления, собранные в `feedback.jsonl`, не используются для улучшения поиска или реранкинга.
8. **Метрики качества чанкера не собираются.** Семантическая когерентность, точность границ, коэффициент перекрытия — все специфицированы, ни одна не измеряется.
9. **Нет фреймворка A/B-тестирования.** Невозможно сравнивать варианты конвейера (например, LangGraph вкл/выкл, разные реранкеры) на одних и тех же запросах.
